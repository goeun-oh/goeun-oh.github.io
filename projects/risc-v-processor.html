32비트 RISC-V ISA 기반 프로세서

Verilog HDL을 활용한 고성능 RISC-V 프로세서 설계

홈 > Projects > 32비트 RISC-V 프로세서
개발 기간 2024.03 - 2024.08
사용 기술
Verilog SystemVerilog RISC-V ISA Vivado ModelSim
팀 구성 개인 프로젝트

프로젝트 개요

본 프로젝트는 RISC-V ISA(Instruction Set Architecture)를 기반으로 한 32비트 프로세서를 Verilog HDL로 설계하고 구현하는 것을 목표로 합니다. 5단계 파이프라인 구조를 통해 성능을 최적화하고, 현대적인 메모리 계층 구조와 캐시 시스템을 포함하여 실제 프로세서 수준의 기능을 구현했습니다.

성능 지표

100 MHz 동작 주파수
5 단계 파이프라인
32 KB L1 캐시
95% 캐시 적중률

주요 기능

🔧 RISC-V ISA 지원

RV32I 기본 명령어 세트와 주요 확장 명령어들을 완전히 지원하여 표준 RISC-V 소프트웨어와 호환됩니다.

⚡ 5단계 파이프라인

Fetch, Decode, Execute, Memory, Writeback 단계로 구성된 파이프라인으로 명령어 처리량을 극대화했습니다.

🏠 메모리 계층 구조

L1 명령어/데이터 캐시와 통합 L2 캐시를 포함한 현대적인 메모리 계층 구조를 구현했습니다.

🔍 분기 예측

2-bit saturating counter 기반의 분기 예측기로 파이프라인 스톨을 최소화합니다.

⚠️ 예외 처리

하드웨어 예외와 인터럽트를 처리하는 완전한 예외 처리 메커니즘을 구현했습니다.

🔧 디버그 지원

JTAG 인터페이스를 통한 디버깅과 성능 모니터링 기능을 제공합니다.

아키텍처 다이어그램

프로세서 전체 아키텍처
RISC-V 프로세서의 전체 블록 다이어그램과 주요 구성요소들의 연결 관계
파이프라인 구조도
5단계 파이프라인의 상세 구조와 데이터 경로, 제어 신호 흐름

기술적 도전과 해결책

파이프라인 해저드

문제: 데이터 의존성과 제어 해저드로 인한 성능 저하
해결: 포워딩 유닛과 분기 예측기 구현으로 스톨 최소화

메모리 지연

문제: 메모리 접근 지연으로 인한 파이프라인 정체
해결: 다단계 캐시 시스템과 프리페치 메커니즘 도입

타이밍 클로저

문제: 복잡한 로직으로 인한 타이밍 제약 위반
해결: 파이프라인 최적화와 클럭 도메인 분리

검증 및 테스트

프로젝트의 정확성과 안정성을 보장하기 위해 다음과 같은 포괄적인 검증 과정을 거쳤습니다:

  • 기능 검증: RISC-V 공식 테스트 스위트를 통한 ISA 호환성 검증
  • 성능 분석: 다양한 벤치마크 프로그램을 통한 성능 측정 및 최적화
  • 시나리오 테스트: 엣지 케이스와 예외 상황에 대한 종합적인 테스트
  • 하드웨어 검증: FPGA 보드에서의 실제 동작 검증
