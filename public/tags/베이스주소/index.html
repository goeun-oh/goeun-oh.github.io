주요 문제점:

💡 핵심 개념 정리

UL 접미사의 의미

UL 접미사를 사용하는 이유:

  • U: Unsigned (부호 없는 정수)
  • L: Long (32비트 이상 보장)
  • 임베디드에서 주소는 항상 양수이므로 unsigned 사용
  • 포인터 연산의 안전성 보장

volatile 키워드의 중요성

volatile을 사용하는 이유:

  • 컴파일러 최적화 방지
  • 메모리 맵드 레지스터는 하드웨어에 의해 값이 변경될 수 있음
  • 매번 메모리에서 값을 읽어와야 함을 보장
